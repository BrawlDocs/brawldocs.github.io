# Stage File System

The **Stage File System** is a system created by DukeItOut that makes stages much more flexible, allowing users to add their own stages and manipulate the stage list, rather than stages using hardcoded IDs.

This system is a series of codes that introduces the ability to use [ASL files](docs/stage-slots?id=asl-files) that map to [param files](docs/stage-slots?id=param-files) to dictate what stages to load. It also allows for a customizable stage roster via either an ASM table or RSS file, depending on the version of the system being used, and ASL files can be used to load alternate stages based on specific button presses while navigating the stage selection screen.

Recent versions of the system also allow users to modify what stages appear on the [Random Stage Switch](docs/random-stage-switch.md) screen and to enable [list alts](docs/stage-slots?id=list-alts) that you can view by pressing certain button combinations in-game.

Most modern builds implement this system.

For detailed information on how to use the system, see [Stage Slots](docs/stage-slots.md) and [Random Stage Switch](docs/random-stage-switch.md).
